# redstr BIS Gap Notes (2026-04-16)

This note captures the current gaps blocking redstr from being best-in-slot.

## 1) CLI contract mismatch with docs (Critical)

- README documents `--list-modes`, `--json`, `--seed`, and `random-user-agent`.
- Current CLI parser does not implement those options/modes.
- `redstr --help` is treated as transform input unless passed as mode.

Evidence:
- `README.md:158-167`
- `crates/redstr/src/main.rs:12-42`

## 2) Incomplete API parity across bindings (Critical)

- Core exports significantly more transformations than Node/Python/WASM.
- FFI surface is also much smaller than core, which blocks .NET parity.

Current counts:
- Core public transform fns: 71
- Node: 46
- Python: 46 (+ module init fn)
- WASM: 46
- FFI: 23

High-impact missing groups in non-Rust bindings:
- Cloudflare/browser-fingerprint functions
- advanced web security helpers (`graphql_*`, `jwt_*`, `http_*`, session/API helpers)
- extra injection variants (`couchdb`, `dynamodb`, `nosql_operator`, `ssti_*`)

Evidence:
- `crates/redstr/src/lib.rs:100-140`
- `bindings/node/src/lib.rs:325-340`
- `bindings/python/src/lib.rs:329-488`
- `bindings/wasm/src/lib.rs:322-337`
- `ffi/src/lib.rs` exported `redstr_*` set

## 3) CI has false-green paths for bindings (High)

- Python build in bindings workflow uses `continue-on-error: true`.
- WASM build in bindings workflow uses `continue-on-error: true`.
- WASM job runs `cargo test -p redstr-wasm`, but current run shows `0 tests`.

Evidence:
- `.github/workflows/bindings.yml:123`
- `.github/workflows/bindings.yml:143`
- `.github/workflows/bindings.yml:147`

## 4) Version/release consistency issues (High)

- Workspace manifest contains an invalid/unused key: `workspace.package.name`.
- .NET package version (`0.2.6`) is behind workspace/core (`0.2.14`).
- Bindings publish pipeline currently validates/publishes npm + PyPI only.

Evidence:
- `Cargo.toml:22`
- `bindings/dotnet/src/Redstr/Redstr.csproj:11`
- `.github/workflows/publish-bindings.yml:55-157`

## 5) Distribution automation is partial (Medium)

- No NuGet publishing workflow.
- WASM distribution path is not wired to npm publish flow as a distinct package pipeline.

Evidence:
- `.github/workflows/publish-bindings.yml`
- `bindings/wasm/`
- `bindings/dotnet/`

## 6) Non-Rust binding test depth is limited (Medium)

- Core tests/clippy are strong.
- FFI has minimal test set.
- Python has pytest config but no `bindings/python/tests` directory currently.
- WASM unit test module exists but not exercised in browser-driven CI.

Evidence:
- `bindings/python/pyproject.toml:45`
- `bindings/python/` (no tests directory)
- `bindings/wasm/src/lib.rs:344-372`

## 7) Trust/readiness artifacts missing (Medium)

- No `SECURITY.md`
- No structured `CHANGELOG.md`
- No explicit MSRV policy (`rust-version`) in crate metadata
